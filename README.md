# QA Quiz Web Application

Веб‑приложение для тестирования знаний по QA с автоматическим сохранением результатов в Google Sheets и локальную базу данных.

## Особенности

- Несколько квизов под разные уровни (Junior / Middle / Senior)
- Гибкая конфигурация тестов через `TESTS_CONFIG` (количество вопросов и теги квизов)
- Автоматическое сохранение результатов в Google Таблицы
- Локальная база данных SQLite для хранения результатов
- Поддержка кириллицы
- Flask backend + Vanilla JS frontend
- Экспорт результатов в CSV для дальнейшего анализа

## Технологии

- **Backend:** Python, Flask, SQLite
- **Frontend:** HTML, CSS, JavaScript
- **Интеграция:** Google Sheets API, gspread
- **Деплой:** GitHub Pages (frontend), локальный Flask‑сервер (backend)

## Установка

1. Клонировать репозиторий:
```bash
git clone https://github.com/pegashevk4-rgb/qa-quiz-web.git
cd qa-quiz-web
```

2. Создать виртуальное окружение:
```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
# или
source .venv/bin/activate  # Linux/Mac
```

3. Установить зависимости:
```bash
pip install -r requirements.txt
```

4. Создать файл `.env` (на основе `.env.example`):
```bash
cp .env.example .env
```

Заполнить `.env` своими данными:
- `SERVICE_ACCOUNT_FILE` — имя JSON‑файла с ключом Google API
- `SPREADSHEET_ID` — ID вашей Google Таблицы (из URL)

5. Настроить Google Sheets API:
   - Создать проект в [Google Cloud Console](https://console.cloud.google.com)
   - Включить Google Sheets API
   - Создать сервисный аккаунт и скачать JSON‑ключ
   - Поместить JSON‑ключ в корень проекта
   - Добавить email сервисного аккаунта (`...@....iam.gserviceaccount.com`) в настройки доступа Google Таблицы как **Редактор**

## Запуск

```bash
python server.py
```

Открыть в браузере: `http://localhost:5000`

Статичный frontend (GitHub Pages) можно настроить так, чтобы он ходил на запущенный backend по API.

## Как это работает

- Пользователь выбирает нужный квиз (например, `QA Junior`, `QA Middle`, `QA Senior`).
- Приложение загружает вопросы, показывает их по одному и собирает ответы.
- По завершении теста результат сохраняется:
  - в локальную SQLite‑базу (`quiz_results.db`);
  - в указанную Google Таблицу (через сервисный аккаунт).
- Для анализа результатов можно:
  - открыть Google Таблицу и работать с фильтрами/графиками;
  - запустить скрипт `export_to_csv.py` или `view_results.py` и получить CSV с результатами.

## Для кого

- **HR-специалисты и рекрутеры.** Быстрая первичная оценка уровня кандидата по QA без сложной настройки и ручной проверки.
- **Тимлиды и руководители команд.** Инструмент для регулярного чек‑апа команды, выявления пробелов в знаниях и подготовки к performance review.
- **Кандидаты и джуны.** Возможность потренироваться перед собеседованием и понять, какие темы по QA стоит подтянуть.
- **Учебные курсы и менторы.** Готовый набор квизов для контрольных работ и домашних заданий по тестированию.

## Сценарии использования

- **Пре-скрининг кандидатов.**  
  HR отправляет кандидату ссылку на нужный квиз (Junior/Middle/Senior), получает результаты в Google Таблицу и быстро отфильтровывает тех, кто не проходит порог.

- **Регулярный чек‑ап команды.**  
  Тимлид запускает периодическое тестирование (например, раз в квартал), сравнивает результаты по датам и видит, как меняется уровень знаний команды.

- **Подготовка к собеседованию.**  
  Кандидат или джун проходит квизы самостоятельно, смотрит на слабые темы и на основе этого формирует план подготовки.

- **Учебные занятия и курсы.**  
  Преподаватель даёт студентам ссылку на квиз как контрольную работу, а затем анализирует результаты прямо в Google Таблице или через экспорт в CSV.
