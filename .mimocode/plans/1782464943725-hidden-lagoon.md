# План: дозированные вопросы по темам

## Суть
Каждое прохождение квиза включает все темы уровня, но из каждой темы берётся N случайных вопросов (по умолчанию 3). HR может менять N. Темы отображаются кандидату.

## Ключевые файлы
- `backend-api/app/models.py` — модели Company, DetailedResult
- `backend-api/app/schemas.py` — QuestionPublic, CompanyPublic
- `backend-api/app/main.py` — get_test_public (строка 443), submit_test (строка 530)
- `quiz/app.js` — загрузка вопросов (строка 68), показ вопроса (строка 176)
- `quiz/index.html` — таймер, интро текст
- `hr-dashboard/index.html` — настройки HR

### 4. Исправить баг в submit_test — total считается по всем вопросам БД
**Файл**: `backend-api/app/main.py:530`

Заменить:
```python

# Стало:

```

## Проверка
1. `GET /public/tests/qa_junior_web?company_token=...` — вопросы сгруппированы по темам, ≤N на тему
2. Пройти квиз — тема отображается, вопросы не повторяются в пределах темы
3. Отправить результат — процент считается от фактического числа вопросов
4. HR Dashboard — изменить N, пройти квиз заново, проверить новое кол-во
